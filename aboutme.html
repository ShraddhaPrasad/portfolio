7/5/17 Python and Scratch

There are many similarities between Python and Scratch. One is the fundamentals of the coding. For example, both can draw shapes based on the code you write. They also both use loops, conditionals, and can use user input to make something new happen each time. Scratch, however, has code already typed up and blocks for you to choose from. Python is slightly more difficult because you have to write all the code by yourself. Syntax plays a bigger part in Python than in Scratch.

Debugging our programs takes more time than actually writing them. Debugging wasn't hard as long as you had a systematic way to go about it. For example, while I was debugging I would test small parts of the code at a time and comment out the rest. That way, I could see exactly what was working and wasn't and narrow down the problem. Once I found where the mistake was it was easy to fix. Debugging can also get tedious, but necessary.
